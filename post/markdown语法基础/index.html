首次接触Markdown是在某个独立博客上，当时在搜一个技术上的问题，结果一进去，就被它的排版吸引了。后果就是想搜的东西没有搜到，却开始了我的Markdown之旅…

找了几个常用到的介绍下，也是学完Markdown的一个小总结吧。

基本操作

  • 斜体/粗体/斜体+粗体

    示例： 斜体 / 粗体 / 粗体+斜体 /
    实现： *斜体* | **粗体**| ***粗体+斜体***

  • 删除线

    示例： 删除线删除线删除线 实现： ~~删除线删除线删除线~~

  • 下划线

    示例： 下划线，需要html标记
    实现： 需要用到HTML语法标记<u>

代办事项

实现方式： 待办- [] / 已办- [x]

高亮代码

@requires_authorization
class SomeClass:
    pass

if __name__ == '__main__':
    # A comment
    print 'hello world'

实现方式：用一组```将代码括起来即可，其中在开头的```后面可以注释是什么代码（如：python），以便得到正确的语法高亮效果。

表格

Markdown语法制表

| 单词 | 翻译 | 例句 | | :———— |:—————-|:——————| |flower | 花 | this is a flower | | user | 用户 | | | password | 密码 | your pasword |

效果如下:

参数 说明 默认值
flower 花 this is a flower
user 用户
password 密码 your password

图片

语法比较简单：![图片描述](地址)即可,图片可以和自己的文章放在一起，也可以放在空间服务商处，将生成的链接放在地址处。

链接

语法和图片几乎一致，只是少了个叹号：[google](http://www.google.com “google”) => google

这个是直接给文字加链接（文字两段加<>即可）：http://example.com/

Markdown编辑器推荐

现在好的Markdown编辑器已经很多，实际用了几个，感觉不错的先放上来：

  • typora：所写即所得，直接，支持语法较全面
  • yu-writer：简洁可定制
  • 作业部落：这个不用多说了，用就是了
  • 马克飞象：Evernote良伴
  • 有道云笔记：虽然还有很多不足，但是已经不错了,文章收藏利器

备注：本文是个摘要，常用的、有遇到过问题的写上来了，以备不时之需，可能有的介绍略简单，不过实际上手试一下就明白了。
